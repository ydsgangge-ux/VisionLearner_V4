# VisionLearner 三种问答模式

## 概述

系统现在支持三种问答模式，可以根据需求选择最适合的模式。

---

## 1. 自动模式（auto）

**描述**：智能混合模式，系统自动判断使用本地知识还是调用LLM

**优先级**：
1. 技能系统（特殊指令，如"测试我"、"进度如何"）
2. 向量库检索（已有知识，相似度 > 0.7 直接返回）
3. 知识树精确查找（精确匹配知识节点）
4. 向量检索 + LLM综合（基于相关知识生成答案）
5. 纯LLM兜底（知识库完全没有时）

**适用场景**：
- 日常学习提问
- 不确定知识库中是否有相关内容
- 希望系统自动优化回答策略

**特点**：
- 智能平衡速度和质量
- 尽可能使用本地知识节省token
- 必要时调用LLM获取新知识

---

## 2. 本地知识模式（local）

**描述**：只从本地知识库检索答案，绝不调用LLM

**优先级**：
1. 技能系统
2. 向量库检索（相似度 > 0.5）
3. 知识树精确查找
4. 没找到：提示"本地知识库中没有找到相关内容"

**适用场景**：
- 复习已学内容
- 测试自己对知识的掌握程度
- 快速查询，不消耗token
- 离线环境或LLM不可用时

**特点**：
- 极快响应速度
- 零token消耗
- 只返回已学内容
- 如知识库中没有，会提示切换到大模型学习模式

---

## 3. 大模型学习模式（llm）

**描述**：直接调用LLM生成详细答案，并自动保存到知识库

**特点**：
- 不检查本地知识库
- 直接调用LLM生成答案
- 答案更详细（4-8句话）
- 自动保存到知识库供后续使用

**适用场景**：
- 学习新知识
- 需要深入、全面的答案
- 扩充本地知识库
- 探索性提问

**特点**：
- 内容最详细
- 自动积累知识
- 一次性学习，永久可用
- 消耗较多token

---

## 使用方式

### Web界面（test_ui.html）

在AI问答区域选择下拉菜单中的模式：

1. 选择"自动模式"（推荐日常使用）
2. 选择"本地知识模式"（复习、测试时使用）
3. 选择"大模型学习模式"（学习新内容时使用）

### API调用

```javascript
// 自动模式
POST /api/ask
{
  "message": "光合作用的原理是什么",
  "mode": "auto"
}

// 本地知识模式
POST /api/ask
{
  "message": "光合作用的原理是什么",
  "mode": "local"
}

// 大模型学习模式
POST /api/ask
{
  "message": "光合作用的原理是什么",
  "mode": "llm"
}
```

### Python代码

```python
from main import LearningSystem

system = LearningSystem()
system.select_goal("goal_id")

# 自动模式
answer = system.answer("光合作用的原理是什么", mode="auto")

# 本地知识模式
answer = system.answer("光合作用的原理是什么", mode="local")

# 大模型学习模式
answer = system.answer("光合作用的原理是什么", mode="llm")
```

## 答案来源标注

所有模式都会在答案前标注来源：

- **[技能系统]** - 技能系统处理
- **[本地知识]** - 从本地知识库检索
- **[本地知识（向量检索）]** - 通过向量检索获得的相关知识
- **[大模型]** - 直接由LLM生成
- **[LLM回答]** - 兜底情况下LLM回答

---

## 推荐使用流程

1. **初始学习阶段**：使用"大模型学习模式"学习新知识
2. **日常提问**：使用"自动模式"，系统智能选择最佳策略
3. **复习测试**：使用"本地知识模式"检验掌握程度
4. **遇到瓶颈**：切换回"大模型学习模式"获取更深入的内容

---

## 注意事项

1. **首次提问**：如果知识库为空，只有"大模型学习模式"才能获得答案
2. **重复提问**：使用"本地知识模式"可以快速复习，避免重复消耗token
3. **流式输出**：所有模式都支持流式输出，体验更好
4. **模式切换**：可以根据实际情况随时切换模式，没有限制
